# PVP 竞技场设计

> 配套文档：`play.md`（PVP 对战）、`battle-protocol.md`（战斗协议）、`tech-stack.md`（排行榜/战斗模块）
> 目标：定义 PVP 竞技场的完整设计，包括匹配机制、段位系统、赛季规则、奖励结构

---

## 一、设计目标

1. **低门槛参与**：异步战报模式，无需双方实时在线，降低匹配等待
2. **战力匹配**：基于防守方阵容战力匹配对手，保证对局基本公平，同时保留养成价值
3. **长线激励**：赛季制 + 段位奖励 + 排行榜，制造持续动力
4. **防作弊**：服务端权威战斗结算，攻击方提交指令 vs 防守方 AI 防守

## 二、战力计算

### 2.1 战力公式

```
单卡战力 = (HP × 0.5 + ATK × 1.0 + MATK × 1.0 + DEF × 0.8 + MDEF × 0.8) × (1 + SPD × 0.01)
阵容战力 = Σ(上场 5 张卡牌的单卡战力)
```

### 2.2 战力用途

| 用途 | 说明 |
|---|---|
| PVP 匹配 | 按防守方阵容战力匹配，战力相近优先 |
| 战前展示 | 匹配到对手后，展示双方基础战力对比 |
| 玩法门槛 | 爬塔/冒险等 PVE 关卡可设推荐战力 |

## 三、核心模式选择

### 推荐方案：异步战报（Async Battle Report）

| 对比维度 | 异步战报 ✅ | 实时同步 |
|---|---|---|
| 服务器压力 | 低（无需长连状态同步） | 高（帧同步/tick 广播） |
| 匹配体验 | 秒级匹配（攻击方即时开打） | 需等待对手确认 |
| 实现复杂度 | 低（复用 PVE 战斗引擎 + AI 防守） | 高（需同步框架） |
| 防作弊 | 简单（服务端全算） | 复杂（需反外挂） |
| 公平性 | 防守方走 AI（略吃亏，通过战力匹配补偿） | 完全公平 |

**结论**：采用异步战报模式。攻击方提交阵容+指令 vs 防守方阵容（由 AI 自动操作），服务端权威结算。

## 四、匹配机制（战力匹配）

### 4.1 匹配流程

```
玩家点击"匹配"
    → 计算攻击方当前阵容战力
    → 搜索防守方池：优先匹配战力 ±5% 范围内的对手
    → 若 3 秒内无结果，扩大到 ±10%
    → 若 8 秒内无结果，扩大到 ±20%
    → 若 15 秒内无结果，不做限制取最近匹配
    → 匹配到后，加载防守方"防守阵容"（离线数据）
    → 进入战斗
```

### 4.2 战力匹配范围

| 匹配耗时 | 战力差异范围 |
|---|---|
| 0-3 秒 | ±5% |
| 3-8 秒 | ±10% |
| 8-15 秒 | ±20% |
| 15+ 秒 | 不做限制 |

**设计意图：** 战力匹配而非 MMR 匹配，确保养成投入始终有价值（战力高的玩家在竞技场中也有优势），同时不会匹配到战力碾压的对手。

### 4.3 防守阵容

- 每位玩家可设置一套**防守阵容**（从已有卡牌中选择）
- 防守阵容的卡牌属性 = 玩家当前卡牌实例的实时属性
- 防守方由 **AI 自动操作**（aggressive/balanced/defensive 策略由配置表设定）
- 防守阵容一经保存即锁定快照，战斗中使用快照属性，避免"换装偷鸡"

## 五、段位与赛季

> 注意：匹配按战力，升段按积分（Score）。积分是独立的排位分数，与战力无关。
> 战力决定"你和谁打"，积分决定"你排第几"。

### 5.1 段位体系

| 段位 | 积分区间 | 说明 |
|---|---|---|
| 青铜 | 0 - 1199 | 初始段位 |
| 白银 | 1200 - 1599 | — |
| 黄金 | 1600 - 1999 | — |
| 铂金 | 2000 - 2499 | — |
| 钻石 | 2500 - 2999 | — |
| 大师 | 3000 - 3599 | 排名显示 |
| 传说 | 3600+ | 全区前 100 名 |

### 5.2 积分变动规则

| 对局类型 | 胜 + | 败 - |
|---|---|---|
| 同段位 | ±25 | -25 |
| 低打高（对手段位高 1 级+） | +35 | -15 |
| 高打低（对手段位低 1 级+） | +15 | -35 |

### 5.3 赛季规则

| 参数 | 值 |
|---|---|
| 赛季长度 | 4 周（28 天） |
| 赛季重置 | 段位软重置（新积分 = 1000 + (旧积分 - 1000) × 0.5） |
| 赛季奖励 | 按最高段位发放（非最终段位） |
| 排行榜 | 实时 Redis Sorted Set，赛季末快照归档 |
| 段位保护 | 大段位不掉段（如黄金 III 0 分不降到白银 I） |
| 连胜加成 | 连胜 5+ 每场额外 +5 积分 |

## 六、战斗规则

### 6.1 基本规则

| 参数 | 值 |
|---|---|
| 上场卡数 | 5 张（2 前 3 后） |
| 操作模式 | 半自动（攻击方可手动触发终极技能） |
| 操作时限 | 30 秒/回合 |
| 回合上限 | 50 回合（超时判攻击方负） |
| 胜负判定 | 一方全员阵亡即结束 |

### 6.2 公平性保障

- **战力匹配**：匹配到的对手战力相近（±5%-20%），避免战力碾压
- **防守方 buff**：防守方获得 HP+10% 加成，补偿 AI 操作劣势
- **阵容克制博弈**：5 卡 5 职业 + 6 元素克制，存在阵容相克，提供非数值取胜空间
- 战力差距在 ±5% 内的对局，胜负主要取决于阵容搭配、元素克制和终极技能释放时机

## 八、结算流程与防作弊

### 8.1 结算流程

```
1. 玩家发起匹配，上传阵容
2. 服务端按战力匹配对手，获取防守阵容快照
3. 进入战斗（BattleStart → BattleAction × N → BattleResult）
4. 服务端权威结算，输出胜负
5. 更新双方积分、段位、胜场记录
6. 异步通知防守方"你被攻击了"（战报可查看）
```

### 8.2 防作弊措施

- 战斗结果服务端全算，客户端仅演出
- 同一对手短时间内不可重复匹配（防刷分）
- 积分变动异常检测（如短时间内 VS 同一对手多次）
- 攻击频率限制（每小时最多 20 场）
