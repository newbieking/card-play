# 战斗系统详细规则

> 配套文档：`play.md`（核心玩法）、`battle-protocol.md`（确定性帧协议）、`tech-stack.md`（战斗校验模块）
> 目标：定义战斗系统的完整规则，包括属性体系、伤害公式、元素克制、状态效果、回合流程，作为客户端/服务端战斗引擎的共同实现依据

---

## 一、基础属性体系

### 1.1 卡牌战斗属性

采用传统 MMO/RPG 标准的**面板分离 + 伤害类型分离**设计：

| 属性 | 标识 | 说明 |
|---|---|---|
| 生命值 | HP | 降至 0 则阵亡，不可复活（除非有复活技能） |
| 物理攻击 | ATK | 影响普攻与物理技能伤害 |
| 法术强度 | MATK | 影响法术技能伤害 |
| 物理防御 | DEF | 减免物理伤害 |
| 法术抗性 | MDEF | 减免法术伤害 |
| 速度 | SPD | 决定每回合行动顺序 |
| 暴击率 | CRIT% | 造成暴击的概率（默认 5%） |
| 暴击伤害 | CDMG% | 暴击时的伤害倍率（默认 150%） |
| 命中率 | HIT% | 攻击命中概率（默认 100%，受闪避抵消） |
| 闪避率 | EVA% | 躲避攻击概率（默认 0%） |
| 吸血率 | LS% | 造成伤害转化为自身治疗的比例（默认 0%） |

### 1.2 卡牌面板属性

| 属性 | 说明 |
|---|---|
| 稀有度 | 普通/稀有/史诗/传说 — 决定基础属性成长曲线 |
| 职业 | 法师/术士/战士/射手/辅助 — 决定技能池与定位 |
| 星级 | 1★~5★（传说可达 10★）— 属性倍率加成 |
| 等级 | Lv1~Lv100 — 基础属性成长 |
| 元素 | 火/冰/暗/光/自然/雷电 — 克制关系 |
| 战力 | 综合战斗力的数值化展示（由全部面板属性加权计算，用于 PVP 匹配） |

**已决策：** ATK/MATK、DEF/MDEF 面板完全分离，物理职业侧重 ATK+DEF，法系职业侧重 MATK+MDEF。

## 二、伤害计算公式

### 2.1 基础伤害公式（经典双曲线防御公式）

采用经典双曲线防御公式，特点：防御无硬上限、收益递减但不封顶、不会出现负防御增伤失控。

```
减伤率(物理) = DEF / (DEF + K)
减伤率(法术) = MDEF / (MDEF + K)
其中 K = 500  （即 500 防御 = 50% 减伤率）
```

```
物理伤害 = ATK × 技能倍率 × (1 + 元素克制加成) × (1 - 物理减伤率)
法术伤害 = MATK × 技能倍率 × (1 + 元素克制加成) × (1 - 法术减伤率)
```

```
暴击伤害 = 最终伤害 × CDMG%
```

```
最终伤害 = ceil(暴击/非暴击伤害 × 随机浮动[0.95, 1.05])  // 定点数整数
最终伤害 = max(最终伤害, 1)  // 保底 1 点伤害
```

### 2.2 伤害类型

| 类型 | 攻击面板 | 防御面板 | 可暴击 | 说明 |
|---|---|---|---|---|
| 物理伤害 | ATK | DEF | 是 | 普攻、物理技能（战士/射手） |
| 法术伤害 | MATK | MDEF | 是 | 法术技能（法师/术士） |
| 真实伤害 | — | 无视双防 | 否 | 固定数值穿透，克制高防目标 |
| 固定伤害 | — | 无视双防 | 否 | 按目标最大 HP 比例，Boss 专属 |

### 2.3 防御收益参考（K=500）

| DEF 值 | 减伤率 | 等效血量倍率 |
|---|---|---|
| 100 | 16.7% | ×1.2 |
| 250 | 33.3% | ×1.5 |
| 500 | 50.0% | ×2.0 |
| 1000 | 66.7% | ×3.0 |
| 2000 | 80.0% | ×5.0 |
| 5000 | 90.9% | ×11.0 |

双曲线公式保证 DEF 从 0→500 收益最高（50%减伤），后续递减但永不封顶，数值策划可放心投放高防御属性。

**预留扩展：** 穿透属性（无视百分比防御）、等级压制系数可在公式外层追加，不影响核心双曲线结构。

## 三、元素/职业克制体系

### 3.1 元素克制

```
火 → 冰 → 雷电 → 水 → 火（循环克制）
光 ⇄ 暗（互克）
自然 → 中立（不被克制，也不克制任何元素）
```

| 克制关系 | 伤害加成 | 受击减伤 |
|---|---|---|
| 克制攻击 | +25% | — |
| 被克制攻击 | — | +15% |

**已决策：** 克制倍率固定，不受稀有度/技能影响。稀有度/技能差异体现在基础面板和技能倍率上。

### 3.2 职业特性（五职业体系）

| 职业 | 定位 | 主属性 | 特性 |
|---|---|---|---|
| 战士 | 前排坦克 | ATK+DEF | HP/DEF 高，速度慢，物理单体强 |
| 法师 | 远程 AOE | MATK | 法术倍率高，HP/DEF 低 |
| 术士 | 控制/削益 | MATK | 施加 debuff，DOT/吸血续航 |
| 射手 | 远程持续输出 | ATK | 高暴击/高速度，需前排保护 |
| 辅助 | 治疗/增益 | MATK | 治疗队友，上 buff/净化 |

**预留扩展：** 刺客（物理爆发/切后排）、骑士（混合坦/团队光环）。

## 四、回合流程

### 4.1 战前准备

```
1. 玩家选择关卡/对手
2. 部署阵容（N 张卡，如 3-5 张）→ 阵型位置
3. 提交阵容至服务端（BattleStart 协议）
4. 服务端下发敌方阵容、种子、先手方（BattleInit 协议）
```

### 4.2 单回合流程

```
┌────────────────────────────────────────┐
│              回合开始                   │
│  1. 回合开始阶段：触发回合型 buff/debuff │
│  2. 行动排序：按 SPD 降序排列           │
│  3. 行动阶段：依次执行各卡牌行动          │
│     ├─ 检查是否存活                     │
│     ├─ 检查是否有控制状态（跳过回合）     │
│     ├─ 执行 AI 自动行动 或 玩家指令     │
│     └─ 结算伤害/状态/反击/死亡           │
│  4. 回合结束阶段：结算 DOT/持续治疗/buff  │
│  5. 检查胜负条件                         │
└────────────────────────────────────────┘
```

### 4.3 行动优先级

```
SPD 高者先动 → 同 SPD 时随机 → 先手方同 SPD 优先
```

- 是否有"先手/后手"全局 buff 补偿？

## 五、技能系统

### 5.1 技能分类

| 类型 | 槽位 | 触发方式 | 说明 |
|---|---|---|---|
| 主动技能 | 技能槽 1 | 自动/手动 | 每个卡牌的主要输出技能 |
| 被动技能 | 技能槽 2 | 自动触发 | 常驻效果或条件触发 |
| 终极技能 | 技能槽 3 | 手动触发 | 需要蓄力/冷却/资源才可释放 |

### 5.2 技能效果类型

| 大类 | 子类型 | 示例 |
|---|---|---|
| 伤害类 | 单体/范围/穿透/溅射 | 冰锥术、暴风雪 |
| 控制类 | 冻结/眩晕/沉默/减速/嘲讽 | 寒冰禁锢 |
| 增益类 | ATK↑/DEF↑/SPD↑/暴击↑/护盾 | 冰霜护盾 |
| 削益类 | ATK↓/DEF↓/SPD↓/易伤/破甲 | 暗影诅咒 |
| 治疗类 | 单体治疗/群体治疗/再生 | 生命之泉 |
| 特殊类 | 复活/反射/吸血/无敌/换位 | — |

### 5.3 技能资源机制（混合制）

**已决策：采用混合制 — 怒气制驱动角色技能，冷却制驱动玩家技能。**

#### 怒气制（角色主动技能）

```
怒气上限：100
获取方式：
  - 普攻：+20 怒气
  - 受击：+10 怒气
  - 击杀：+30 怒气
  - 部分被动技能可额外产怒

消耗规则：
  - 每个卡牌的主动技能（技能槽 1）消耗 40-100 怒气（由技能配置表决定）
  - 怒气足够时 AI 自动释放（自动战斗）或玩家手动触发（半自动）
  - 战斗开始时初始怒气 = 0（或受某些被动技能影响）
```

#### 冷却制（玩家全局终极技能）

```
机制：
  - 每个卡牌的终极技能（技能槽 3）有独立冷却回合数（由技能配置表决定，如 4-8 回合）
  - 冷却从战斗开始即计数
  - 冷却完毕后出现"可释放"提示，玩家手动确认释放
  - 释放后进入下一轮冷却
  - 不受怒气系统影响

混合制的优势：
  - 怒气制：制造节奏感，普攻/受击/击杀→蓄满→释放→循环
  - 冷却制：终极技能有明确的释放窗口，增强玩家决策权重
  - 两者互不干扰，战斗策略层次更丰富
```

## 六、状态效果（Buff/Debuff）

### 6.1 状态类型定义

| 状态 | 类型 | 效果 | 持续回合 | 可叠加 |
|---|---|---|---|---|
| 冰冻 | 硬控 | 跳过行动，受击+50%伤害后解冻 | 1-2 | 否 |
| 眩晕 | 硬控 | 跳过行动 | 1 | 否 |
| 沉默 | 软控 | 无法使用主动/终极技能 | 1-2 | 否 |
| 减速 | 软控 | SPD -30% | 2-3 | 否（刷新覆盖） |
| 中毒 | DOT | 每回合末损失施法者 ATK×20% 的 HP | 3 | 可叠加（层数） |
| 灼烧 | DOT | 每回合末损失固定伤害 | 2-3 | 可叠加 |
| 护盾 | Buff | 吸收伤害 | 永久（至被击破） | 刷新覆盖 |
| ATK/DEF Buff | Buff | 属性 ±X%（乘算或加算） | 2-3 | 否（刷新覆盖） |
| 免疫 | Buff | 免疫指定控制 | 1-2 | 否 |
| 无敌 | Buff | 不受任何伤害 | 1 | 否 |
| 复活 | Buff | 死亡时以 X% HP 复活（一次性） | 永久 | 否 |

### 6.2 状态结算优先级

```
结算顺序：沉默(屏蔽技能) → 眩晕(跳过行动) → 冰冻(跳过行动+破冰) → 其他
DOT 结算：回合结束阶段统一结算，按施加顺序
```

- 护盾是否受防御减免？还是独立吸收？
- 是否有"净化"技能可清除 debuff？

## 七、阵容与阵型

### 7.1 阵型槽位

**已决策：统一 5 张卡牌出战，采用 2 前 3 后阵型。**

```
┌──────────────────────────────┐
│  [前排1]     [前排2]          │  ← 前排：坦克/战士优先站位
│  [后排1]  [后排2]  [后排3]    │  ← 后排：输出/辅助/控制
└──────────────────────────────┘
```

- 前排受击概率 +30%（普攻/AI 优先攻击前排）
- 后排不会被单体近战攻击直接选中（除非前排全灭或有"狙击"/"贯穿"技能）
- 部分 AOE 技能区分"前排范围"和"后排范围"

### 7.2 出战人数

| 模式 | 出战人数 | 备注 |
|---|---|---|
| 冒险 | 5 张 | 统一 5 张，降低不同模式切换的学习成本 |
| 爬塔 | 5 张 | — |
| PVP | 5 张 | — |

**预留扩展：** 阵型激活效果（如"前后排各 2 名同职业 +10% ATK"可在后续版本作为赛季机制引入）。

## 八、战斗结束与奖励

### 8.1 胜负判定

| 结果 | 条件 |
|---|---|
| 胜利 | 敌方全员阵亡 |
| 失败 | 我方全员阵亡（或超时未胜利，如 PVP 限 50 回合） |
| 平局 | 双方同时全灭 |

### 8.2 星级评价（冒险关卡）

| 星级 | 条件 |
|---|---|
| ★★★ | 全员存活通关 |
| ★★☆ | 1 名阵亡 |
| ★☆☆ | 2 名及以上阵亡 |
