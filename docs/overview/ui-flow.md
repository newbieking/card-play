## 二、核心界面详细设计

### 2.1 启动/登录界面

```
┌─────────────────────────────┐
│                             │
│     [游戏 Logo]             │
│                             │
│     [背景动画]              │
│                             │
│  ┌─────────────────────┐   │
│  │    游客登录          │   │
│  └─────────────────────┘   │
│  ┌─────────────────────┐   │
│  │    微信登录          │   │
│  └─────────────────────┘   │
│  ┌─────────────────────┐   │
│  │    Apple 登录        │   │
│  └─────────────────────┘   │
│                             │
│  版本号: v1.0.0             │
│  服务协议 | 隐私政策        │
└─────────────────────────────┘
```

**流程：**
1. 启动 → 检查本地 Token
2. Token 有效 → 直接进入主城
3. Token 无效 → 显示登录界面
4. 游客登录 → `POST /api/v1/account/guest_login`
5. 登录成功 → 获取档案 + 配置版本 → 进入主城

## 三、界面流转规则

### 3.1 返回逻辑

| 当前界面 | 返回目标 | 说明 |
|---|---|---|
| 任何二级界面 | 上一级 | 标准返回 |
| 主城 | 退出确认弹窗 | 防误触退出 |
| 战斗中 | 暂停菜单（不能直接退出） | 战斗中退出判负 |
| 结算界面 | 主城 | 固定返回主城 |

### 3.2 红点提示规则

| 位置 | 红点条件 |
|---|---|
| 主城-任务 | 有可领取的任务奖励 |
| 主城-邮件 | 有未读邮件 |
| 主城-抽卡 | 新手池可抽 / 有免费抽卡券 |
| 主城-PVP | 今日有免费挑战次数 |
| 阵容-卡牌 | 有可升星的卡牌（碎片足够） |
| 阵容-装备 | 有可强化的装备（金币足够） |

### 3.3 弹窗优先级

```
系统公告 > 维护公告 > 活动弹窗 > 红点提示
```

## 四、界面资源清单

### 4.1 核心界面（M1 必需）

| 界面 | 关键素材 | 资源路径 |
|---|---|---|
| 登录 | Logo、背景、按钮 | `ui/login/` |
| 主城 | 背景、图标×8、顶栏 | `ui/hub/` |
| 阵容 | 阵位×5、卡牌列表 | `ui/formation/` |
| 卡牌详情 | 立绘框、属性面板、技能栏 | `ui/card_detail/` |
| 战斗 | 战场背景、血条、技能按钮 | `ui/battle/` |
| 结算 | 奖励展示、星级、按钮 | `ui/result/` |

### 4.2 通用素材

| 素材 | 说明 | 资源路径 |
|---|---|---|
| 按钮 | 通用按钮（主按钮/次按钮/禁用） | `ui/common/btn_` |
| 弹窗 | 通用弹窗框架 | `ui/common/popup_` |
| 图标 | 资源图标（金币/钻石/体力等） | `ui/common/icon_` |
| 红点 | 红点提示 | `ui/common/red_dot` |
| 星级 | 星星图标（空/满） | `ui/common/star_` |
| 稀有度边框 | 普通灰/稀有蓝/史诗紫/传说金 | `ui/common/frame_` |
