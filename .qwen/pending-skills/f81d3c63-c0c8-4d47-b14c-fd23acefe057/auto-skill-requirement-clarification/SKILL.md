# 需求澄清技能

本技能提供在开始实现前澄清需求的方法，确保完全理解用户意图，避免返工和误解。

## 使用场景

- 用户提出新功能需求
- 需求描述模糊或不完整
- 需要确认技术实现细节
- 涉及多个利益相关者

## 澄清步骤

### 1. 理解用户意图

**核心问题**：用户真正想要什么？

**方法**：
1. **复述需求**：用自己的话复述用户的需求
2. **确认目标**：明确最终要达成的效果
3. **识别隐含需求**：发现用户没有明确说出来的需求

**示例**：
```
用户：我想要一个登录功能
澄清：您需要的是用户名/密码登录，还是包括第三方登录（微信、GitHub等）？
```

### 2. 明确范围和边界

**需要澄清的维度**：

#### 功能范围
- 这个功能包含哪些子功能？
- 哪些是必须实现的，哪些是可选的？
- 是否有依赖的其他功能？

#### 技术约束
- 使用什么技术栈？
- 有性能要求吗？
- 需要兼容哪些平台/浏览器？

#### 时间和资源
- 期望的完成时间？
- 有哪些可用的资源？
- 是否有外部依赖？

**示例**：
```
用户：实现用户注册功能
澄清：
1. 注册字段：用户名、密码、邮箱、手机号？
2. 验证方式：邮箱验证、短信验证？
3. 密码强度要求？
4. 是否需要第三方登录集成？
```

### 3. 确认验收标准

**验收标准模板**：
```
当 [条件] 时，[角色] 可以 [操作]，预期结果是 [结果]
```

**示例**：
```
当用户输入有效的邮箱和密码时，用户可以点击注册按钮，预期结果是：
1. 创建用户账户
2. 发送验证邮件
3. 显示注册成功提示
4. 跳转到登录页面
```

### 4. 识别风险和假设

**风险识别**：
- 技术风险：是否有不熟悉的技术？
- 依赖风险：是否依赖外部服务？
- 时间风险：时间是否充足？

**假设确认**：
- 用户有哪些隐含假设？
- 这些假设是否成立？
- 如果假设不成立会怎样？

**示例**：
```
风险：第三方登录 API 可能有调用限制
假设：用户已经有微信开放平台账号
备选方案：如果微信登录不可用，支持邮箱注册
```

## 澄清技巧

### 1. 使用开放式问题

**避免**：是/否问题
**推荐**：开放式问题

```
不好的：您需要登录功能吗？
好的：您希望用户如何登录？
```

### 2. 提供选项

当需求不明确时，提供具体选项：

```
您希望使用哪种数据库？
A. PostgreSQL（推荐，支持复杂查询）
B. MySQL（简单易用）
C. MongoDB（灵活，适合文档存储）
```

### 3. 使用可视化

对于复杂需求，使用图表或原型：

```
这是登录页面的布局，您看是否符合预期？
[显示线框图]
```

### 4. 分阶段确认

将大需求拆分成小部分，逐步确认：

```
第一阶段：基础登录（用户名/密码）
第二阶段：第三方登录（微信、GitHub）
第三阶段：多因素认证
您希望先实现哪个阶段？
```

## 常见澄清场景

### 场景1：功能需求模糊
**用户**：我想要一个搜索功能
**澄清**：
1. 搜索什么内容？（商品、文章、用户？）
2. 搜索范围？（全文搜索、标题搜索、标签搜索？）
3. 搜索结果展示？（列表、卡片、摘要？）
4. 性能要求？（响应时间、并发量？）

### 场景2：技术实现不确定
**用户**：实现一个实时通知功能
**澄清**：
1. 通知类型？（系统通知、消息通知、活动通知？）
2. 实时性要求？（秒级、分钟级？）
3. 通知渠道？（站内信、邮件、短信、推送？）
4. 持久化要求？（是否需要保存通知历史？）

### 场景3：涉及多方利益
**用户**：优化页面加载速度
**澄清**：
1. 哪个页面？（首页、商品页、搜索页？）
2. 当前加载时间？（目标是多少？）
3. 优化方向？（前端优化、后端优化、网络优化？）
4. 预算限制？（是否有服务器升级预算？）

## 沟通技巧

### 1. 积极倾听
- 不打断用户
- 用点头、"嗯"等表示在听
- 复述确认理解

### 2. 确认理解
- "我理解的是...，对吗？"
- "您是说...，我理解正确吗？"

### 3. 管理预期
- 明确说明能做到什么
- 诚实说明做不到什么
- 提供替代方案

### 4. 记录确认
- 重要确认要书面记录
- 发送会议纪要或确认邮件
- 保持沟通可追溯

## 工具辅助

### 1. 原型工具
- Figma、Sketch：界面原型
- Excalidraw：草图绘制
- Mermaid：流程图

### 2. 文档工具
- Markdown：轻量级文档
- Confluence：团队知识库
- Notion：多功能文档

### 3. 沟通工具
- 即时通讯：快速澄清
- 视频会议：复杂需求讨论
- 邮件：正式确认

## 总结

需求澄清是项目成功的关键步骤。通过系统化的澄清流程，可以：
1. **减少返工**：确保理解正确再开始
2. **提高效率**：避免做无用功
3. **控制范围**：明确边界，防止范围蔓延
4. **管理预期**：让各方对结果有合理预期

记住：**花在澄清上的时间永远值得**。磨刀不误砍柴工！
